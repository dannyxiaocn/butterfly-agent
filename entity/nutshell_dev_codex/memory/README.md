# `entity/nutshell_dev_codex/memory`

Codex-specific layered memory for the project-development entity.

## Files

- `track_sop.md`: concise task-execution workflow.
- `work_state.md`: current-task and last-completed task note.
- `monitoring.md`: when and how to use the repo's `.claude/` monitoring scripts deliberately.

## How It Contributes To The Whole System

These files give Codex sessions their own durable operating memory instead of relying on the Claude-oriented memory path used by `nutshell_dev`.
