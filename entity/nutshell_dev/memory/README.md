# `entity/nutshell_dev/memory`

Layered memory for repo-development sessions.

## Files

- `track_sop.md`: step-by-step workflow for implementing and marking `track.md` tasks.
- `work_state.md`: short durable note about the current or last completed task.

The main `memory.md` in the entity root carries broader identity and workflow guidance.

## How To Use This Part

Update these files when you want future `nutshell_dev` sessions to inherit better repo-specific operating memory.

## How It Contributes To The Whole System

These layers seed every new `nutshell_dev` session and help the agent resume long-running repo work consistently.
